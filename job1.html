About The Project

Our client is a startup company providing music-related services on the internet. The main service is a vinyl platform between artists and customers. Platforms Artists can create vinyl records on demand directly for their fans. Worldwide customers ( US/UK/EU/Japan).

Small Team

2 RoR Devs from Mobilunity side + potentially 1 markup engineer;

4 Customer support staff;

1 Engineer (only CTO) / 1 Designer / 1 part-time junior markup engineer.

Current Server Infrastructure And Development Environment

Using AWS for overall of our services server infrastructure;

Using EC2 mainly for web / application server, and using auto scaling group and application load balancer to increase availability and fit to increase and decrease transactions;

Realize deploy automation with above server configuration, we use AWS CodeDeploy service to deploy application code to EC2 instance;

Deployment process (on CodeDeploy) is called from CircleCI automatically, after build and test process passed;

The application is based on ruby on rails (5.1.1);

Web server (nginx), App server (unicorn), job worker (sidekiq) are running on the same EC2 instance and 3-8 these instances are running under the load balancer usually.

The Client Also Use Other AWS Services Below

RDS (MySQL, for storing database record);

ElastiCache (redis, for storing session info, job queue data, temporary application layer cache data, etc.);

S3 (many points, file upload from users, distribution static files (with cloudfront), etc.);

Cloudfront (to increase loading performance for static files);

Route53 (manage domain and DNS);

Started using Docker for development environments;

EC2 instances middleware setup script is written in itamae.

The Problems The Client Is Concerned About

Unstable servers;

Not flexible preview environment;

Complex deploy process;

Difficult troubleshooting;

Old middlewares.

We believe we need someone full-time for a short term (2-3 months) (maybe more, depending on the seniority of DevOps) for sure, and then MAYBE switching to part-time for support (controlling deploys, fixing, etc).
